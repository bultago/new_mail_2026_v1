### 보고 체계
```
개발자 → 모듈 리더 → 기술 리더 → PM → 경영진
   ↓         ↓            ↓        ↓
 일일      주간         월간     분기
```

---

## 정기 회의

### 1. 일일 스탠드업 (Daily Standup)
- **시간**: 매일 오전 10:00
- **소요**: 15분
- **참석자**: 전체 개발팀
- **형식**: 각자 3가지 공유
  1. 어제 한 일
  2. 오늘 할 일
  3. 블로킹 이슈

### 2. 주간 진행 회의
- **시간**: 매주 월요일 오전 10:30
- **소요**: 1시간
- **참석자**: PM, 기술 리더, 모듈 리더, QA 리더, DevOps
- **안건**:
  - 지난 주 완료 작업 리뷰
  - 진행률 확인
  - 이슈 및 위험 검토
  - 이번 주 작업 계획
  - 리소스 조정 논의

### 3. 월간 리뷰 (Monthly Review)
- **시간**: 매월 첫째 주 금요일 오후 2:00
- **소요**: 2시간
- **참석자**: 전체 팀 + 이해관계자
- **안건**:
  - Phase 진행 상황
  - 성과 측정 (완료 작업, 품질, 성능)
  - 위험 및 이슈 리뷰
  - 다음 달 계획
  - 이해관계자 보고

### 4. 임시 회의 (Ad-hoc Meeting)
- **소집**: 필요시 즉시
- **대상**: 관련자만
- **목적**: 긴급 이슈 해결, 기술적 의사결정

## 보고 체계

### 1. 주간 진행 보고서 (Weekly Report)
- **작성자**: PM
- **제출일**: 매주 금요일 오후 6:00
- **수신자**: 경영진, 이해관계자
- **내용**:
  - 완료된 작업 수 (작업 ID 포함)
  - 진행률 (Phase별)
  - 이슈 및 위험
  - 다음 주 계획

### 2. 월간 종합 보고서 (Monthly Report)
- **작성자**: PM + 기술 리더
- **제출일**: 매월 말일
- **수신자**: 경영진
- **내용**:
  - Phase 진행 현황
  - 성과 측정 (품질, 성능, 일정)
  - 예산 집행 현황
  - 위험 관리 현황
  - 다음 달 계획

### 3. Phase 완료 보고서
- **작성자**: PM + 기술 리더
- **제출일**: 각 Phase 완료 시
- **수신자**: 경영진, 전체 팀
- **내용**:
  - Phase 목표 달성 여부
  - 완료 기준 충족 확인
  - 교훈 (Lessons Learned)
  - 다음 Phase 준비 상태

## 커뮤니케이션 도구

- **채널 구성**:
  - `#migration-general` - 전체 공지

- **라벨 체계**:
  - `phase-0` ~ `phase-6`: Phase 구분
  - `bug`: 버그
  - `enhancement`: 개선
  - `question`: 질문
  - `blocked`: 차단됨

### 3. 문서 (Confluence / Wiki)
- **공간 구성**:
  - 마이그레이션 개요
  - Phase별 작업 문서
  - 회의록
  - FAQ

### 4. 화상회의 (Zoom / Teams)
- 주간 회의 및 월간 리뷰
- 화면 공유로 진행 상황 시연

## 정보 공유 방식

### 1. 진행 상황 공유
- **주간**: `docs/plans/README.md` 진행률 대시보드 업데이트
- **월간**: 월간 보고서 배포
- **즉시**: Slack `#migration-general` 채널에 중요 업데이트 공지

### 2. 이슈 공유
- **발견**: 이슈 트래커에 즉시 등록
- **심각**: Slack에서 @channel 멘션
- **해결**: 이슈 트래커 업데이트 + Slack 공유

### 3. 문서 공유
- **작성**: Confluence/Wiki에 업로드
- **공지**: Slack에 링크 공유
- **중요**: 회의에서 설명

## 이해관계자 커뮤니케이션

### 경영진
- **빈도**: 월 1회
- **방식**: 월간 보고서 + 프레젠테이션
- **내용**: 진행률, 성과, 예산, 위험

### 사용자 대표
- **빈도**: 분기 1회
- **방식**: 데모 + Q&A
- **내용**: 주요 변경사항, UI/UX 개선

### 운영팀
- **빈도**: 필요시
- **방식**: 회의 + 문서
- **내용**: 배포 계획, 운영 변경사항
